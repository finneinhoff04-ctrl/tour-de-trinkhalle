SETUP
Nur du als GM machst das – einmal, fertig.
👋 Hast du schon eine Supabase-Instanz? Dann direkt zu Schritt 5. Neu aufsetzen? Dann hier entlang:
1
Account erstellen

Geh auf supabase.com → „Start your project" → kostenlos registrieren.

2
Neues Projekt

„New project" → Name: tour-trinkhalle → Region: EU (Frankfurt) → Create.

3
Tabellen anlegen

SQL Editor → „New query" → einfügen und ausführen:

create table teams ( id bigserial primary key, code text unique not null, name text not null, members jsonb default '[]', route text not null, points integer default 0, completed_stations jsonb default '[]', registered_at bigint ); create table submissions ( id bigint primary key, team_code text not null, station_id text not null, submitted_at bigint, points integer default 0, note text default '' ); create table settings ( key text primary key, value text not null default '' ); alter table teams enable row level security; alter table submissions enable row level security; alter table settings enable row level security; create policy "open" on teams for all using (true) with check (true); create policy "open" on submissions for all using (true) with check (true); create policy "open" on settings for all using (true) with check (true);
4
API Keys holen

Project Settings → API → Project URL + anon public key kopieren.

5
Hier eintragen & verbinden
💡 Netlify Deploy: Diese Datei einfach per Drag&Drop auf app.netlify.com/drop ziehen – fertig. Kein Build, kein Config.
